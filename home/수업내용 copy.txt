<html 기본>
 주석. (단축키 Ctrl + /) 
 이 사이에 있는 모든 내용은 코드에 영향을 미치지 않음
 
 HTML 문서는 태그로 구성
 태그는 다음과 같은 형태로 되어있음
 - 단일 태그 : <태그명 속성="값" 속성="값">
 - 쌍 태그 : <태그명 속성="값" 속성="값"></태그명>
 - 속성은 모든 태그가 가질 수 있는 공통 속성과 태그마다 종류가 다른 속성이 있음
 - 속성의 값은 속성마다 정해져 있는 경우도 있고, 정해져 있지 않은 경우도 있음
 - 값은 앞뒤로 '' 또는 ""을 붙임. 생략도 가능한데 이 경우 값에 공백이 있으면 안됨
 - VS code 에서는 태그를 입력할 때 태그명 입력 후 엔터를 치면 쉽게 태그를 입력가능

<a태그>
 a태그
 - 페이지를 이동할 때 사용하는 태그
 - 기본 값
  - 글자색 : 방문 전 파랑, 방문 후 - 보라, 클릭 중 - 빨강
  - 밑줄 : 있음
 - 속성
  - href : 이동할 위치를 지정
   - URL 주소 입력(http나 https 포함) : 다른 사이트로 이동
   - URL 주소 입력 : 같은 사이트 다른 페이지로 이동
   - tel:전화번호 : 전화번호 연결 - 모바일에서
   - mailto:이메일 주소 : 이메일 작성으로 연결
   - #아이디명 : 같은 페이지에서 다른 위치로 이동 

  - target : 링크 페이지가 표시될 위치
   - _self : 기본값, 현재창
   - _blank : 새창이나 새탭
   - _parent : 부모 프레임
   - _top : 최상위 조상 프레임

  - download : 파일을 다운로드할 때 사용
   - 값을 지정하지 않으면 서버에 있는 파일명을 그대로 다운로드
   - 값에 파일명을 지정하면 지정된 파일명으로 다운로드
  
  - rel : 현재 문서와 링크된 문서의 관계를 나타내는 속성
   - noopener : 링크된 문서가 현재 문서를 참조하지 못하게 함 - 보안 강화
   - noreferrer : 링크된 문서에 출처를 전달하지 않음 - 보안 강화
   - nofollow : 검색 엔진이 링크를 따라가지 않도록 지시 : SEO(검색 엔진 최적화)
   - external : 외부 링크
   - author : 저자
   - help : 도움말
   - license : 라이센스 정보
 
 경로 지정 방식
 - 절대 경로
  - 최상위 폴더를 기준으로 위치를 잡음
  - 최상위 폴더가 변경되면 기존 경로를 변경해야할 수 있음
 - 상대 경로
  - 현재를 기준으로 위치를 잡음
  - 최상위 폴더가 변경되도 기존 경로가 바뀌지 않음
  - ../ : 상위 폴더

 img 태그
 - 이미지를 넣을 때 사용
 - inline-block 태그
 - 속성
  - src : 가져올 이미지 주소
  - alt : 이미지 로딩 실패 시 보여줄 문구이자 스크린 리더기가 읽을 문구
  - width : 이미지 너비. 기본값은 원본 이미지 너비의 100%, auto 지정 가능
  - height : 이미지 높이. 기본값은 원본 이미지 높이의 100%, auto 지정 가능
 - width/height의 auto 사용법
  - width나 height 중 하나를 고정으로 하고, 나머지를 auto로 설정해서 이미지 비율이 깨지지 않게 함

<form태그>  
 form태그
 - 정보를 서버에 전송하는 태그 
 - 속성 
  - action : 전송할 주소 
  - method : 전송할 방식 
   - get : 기본값. URL에 정보를 추가, URL에 정보가 노출 
   - post : header에 정보를 추가, URL에 정보가 노출되지 않음 
  - enctype : 서버에 전송할 정보의 종류 
   - multipart/form-data : 첨부파일을 서버에 전송할 때 사용 

 input 태그 
 - 정보를 입력하거나 선택할 때 사용하는 태그 
 - type에 따라 종류가 달라짐 
 - 속성 
  - type : input 태그의 종류를 지정 
   - text : 문자열 입력칸 
   - password : 비밀번호 입력칸 
   - radio : 여러 개 중 반드시 하나만 선택해야 하는 경우 
   - checkbox : 여러 개를 선택 할 때 사용 
   - date, datetime-local : 날짜/날짜시간을 선택할 때 사용 
   - tel : 전화번호를 입력할 때 사용. 모바일에 최적화 
   - email : 이메일을 입력할 때 사용. 이메일 주소 형식에 맞는지 확인 
   - file : 첨부파일을 선택할 때 사용 
   - number : 숫자만 입력 가능, 화살표가 생김 
   - color : 색상 정보를 선택할 때 사용 
   - button, submit, reset : 일반 버튼/전송 버튼/리셋 버튼 button태그로 대체 
   - image : 이미지를 선택할 때 사용. img태그로 대체 
  - value : input 태그의 초기값을 설정 
  - readonly : 읽기 전용, 수정 불가능, 서버에 전송 가능 
  - disabled : 수정 불가능, 서버에 전송 불가능 
  - required : 필수 항목을 지정. 입력안하고 전송 시 안내문구 나옴 
  - placeholder : 입력 전 안내문구. 문구를 입력하면 사라짐 
  - name  
   - 서버로 정보를 전송할 때 전송할 이름을 지정. 
   - name이 없으면 서버로 전송이 안됨 

 fieldset 태그 
  - form태그 안에서 입력 태그들을 묶을 때 사용

<폰트 속성>
 color
 - 글자 색상
 
 색상 지정 방식
 1. 미리 지정된 색상 이름
  - red, blue, green 등과 같이 100개가 넘는 이름

 2. 16진수로 표현
  - #rrggbb 형태로 표시
  - rr은 빨강 색상, gg는 녹색 색상, bb는 파랑 색상 정보를 16진수로 표현(0~9,a~f)
  - 각 rr값이 같고, gg값이 같고, bb값이 같으면 #rgb로 줄여 쓸 수 있음
  - red => #ff0000 => #f00
  - white => #ffffff => #fff
  - black => #000000 => #000

 3. rgb함수/rgba함수
  - rgb(숫자,숫자,숫자), rgba(숫자,숫자,숫자,투명도)
   - 숫자는 0~255
    - 투명도는 0~1사이의 실수. 1이면 불투명, 0이면 투명
  - #f00 => rgb(255, 0, 0) => rgb(255, 0, 0, 1)

 4. hsl(숫자,숫자,숫자), hsla(숫자,숫자,숫자,투명도)
  - 색상, 채도, 명도를 이용하여 색상을 표시 
  
 font-weight
  - 폰트 굵기
  - 100~900사이의 값을 100단위로 입력
  - normal : 400
  - bold : 700
 
 font-size
  - 폰트 크기
  - 숫자와 단위를 함께 사용
 
 font-family
  - 폰트 종류를 지정
  - 일반적으로 여러 폰트를 한번에 지정
   - 왼쪽 폰트가 있으면 적용, 없으면 오른쪽 폰트로 이동
   - 보통 제일 오른쪽은 기본 폰트 중 하나를 선택(monospace, serif, sans-serif)
  - 폰트 이름이 한글인 경우 영문명도 함께 작성
  - 폰트 이름이 여러 단어인 경우 ''나 ""를 반드시 붙여야 함

<영역 속성>
 box-sizing
 - width/height의 기준을 정하는 속성
 - 값
  - border-box : 너비/높이가 컨텐츠 + 패딩 + 테두리를 합친 width/height
  - content-box : 기본갑, 너비/높이가 컨텐츠의 width/height

 - 요소의 너비/높이를 지정
 - box-sizing 값에 따라 의미가 달라짐
 
 margin
 - 요소의 테두리 밖 여백을 지정하는 속성
 - margin-top, margin-right, margin-bottom, margin-left를 줄여 쓴 단축 속성
 - 방향은 위부터 시계방향. 상 > 우 > 하 > 좌
 - 액자의 위치를 설정
 - 사용 예시
  - margin : 10px 20px 30px 40px
   - 상 : 10px, 우 : 20px, 하 : 30px, 좌 : 40px
  - margin : 10px 20px 30px => 좌우가 대칭
   - 상 : 10px, 우 : 20px, 하 : 30px, 좌 : 20px
  - margin : 10px 20px => 상하 동일, 좌우 동일
   - 상 : 10px, 우 : 20px, 하 : 10px, 좌 : 20px
  - margin : 10px => 상하좌우 동일
   - 상 : 10px, 우 : 10px, 하 : 10px, 좌 : 10px
  - margin : 0 auto
   - 위아래 0, 좌우 자동 배치 => width가 고정px일 때 가운데 배치
 
 padding
 - 요소의 테두리 안 여백을 지정하는 속성
 - 액자안의 사진 시작 위치를 지정
 - 사용 방식은 margin과 동인
 - padding-top, padding-right, padding-bottom, padding-left를 줄여쓴 단축 속성

<float 속성>
 float
 - 요소를 왼쪽, 오른쪽으로 배치할 때 사용하는 속성
 - inline, block에 상관 없이 왼쪽, 오른쪽으로 배치
 - 엔터에 의한 공백을 없애줌
 - 요소에 width를 지정하지 않으면 요소 안에 있는 내용만큼 너비가 잡힘 
 - 부모 요소의 높이가 잡히지 않음
 
 clear
 - float 속성이 적용된 요소가 있으면 다음 요소에 영향을 미치는데 이를 초기화 하는 속성
 - 값
  - left : 왼쪽 float 요소의 높이를 계산한 후 배치
  - right : 오른쪽 float 요소의 높이를 계산한 후 배치
  - both : 왼쪽과 오른쪽 float 요소의 높이 중 긴 요소를 기준으로 배치

 none : 안보이게 감춤
 각 태그 마다 기본 display값이 있는데 이를 변경하는 속성

<position 속성>
 position
 요소를 배치하는 방식을 지정하는 속성
 값
 static : 기본값
 fixed
 요소를 브라우저를 기준으로 위치를 고정
 위로가기 버튼
 top, bottom, left, right 속성을 이용하여 위치를 조절
 + 값이면 요소 안쪽으로 향하고, - 값이면 요소 바깥쪽으로 향함
 relative
 요소의 상대위치를 계산해서 배치
 요소가 원래 있어야할 위치에서 top, bottom, left, right 속성값에 따라 이동
 자손 요소 중 position:absolute인 요소의 기준점이 된다
 absolute
 기준 요소를 기준으로 배치
 top, bottom, left, right 속성을 이용하여 위치를 조절
 기준 요소
 absolute가 적용된 요소의 조상 요소 중 
 position:relative나 position:absolute인 요소가 있는 가장 가까운 조상이 기준요소
 기준 요소가 없으면 브라우저가 기준
 top, bottom, left, right를 이용하여 width와 height를 대신할 수 있음
 absolute가 적용된 요소는 부모 요소 높이에 영향을 주지 않음

<z-index 속성>
 z-index
 요소들이 서로 겹쳤을 때 누구를 위로 보낼지 결정하는 속성
 숫자가 클수록 위로 올라감
 숫자가 같으면 html코드 순서상 뒤에 있는게 위로 올라감(형제 요소들인 경우)
 요소들은 부모 요소를 넘을 수 있음

<opacity 속성 예제>
 opacity
 요소의 투명도를 설정
 0~1사이의 실수
 0이면 투명, 1이면 불투명
 부모 요소에 투명도가 적용되면 자식 요소에도 적용됨
 자식 요소에 투명도를 1로 해도 투명이 됨
 배경 색을 rgba로 이용하는게 좋을 수도 있음
