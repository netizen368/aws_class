변수

자료형 변수명;

  - 기본형 : 값을 저장
  - 참조형 : 주소를 저장 => ==와 != 비교 금지 => 객체의 경우 equals
- 변수명 작성 규칙/관례

연산자 
- 대입 : = 
- 산술 : +, -, *, /, % => 정수/정수=>정수
- 비교 : <, >, <=, >=, !=, ==
- 논리 : &&, ||, !
- 증감 : ++, -- => 최종적으로 1증가/1감소
- 조건 : 조건식?참 : 거짓;
- 우선순위 : ()

조건문
- A이면 B해라
- A 조건식, B 실행문
- 모든 조건문은 if문으로 표현 가능하다
- if문
- switch문

반복문
- 규칙적인 작업을 반복적으로 할 때 사용
- for문, while문, do while문
- break, continue

배열
- 여러 자료를 한번에 관리
- 반복문을 이용할 수 있다
- 배열 선언 방법
- 배열은 0번지, 크기-1번지까지

객체 지향 프로그래밍 언어의 특징
 - 추상화 : 현실 세계의 정보를 코드로 만드는 것
 - 캡슐화 : 필요한 정보를 클래스로 만들고,
          사용자가 필요한 기능만 접근 가능하게 하는 것
 - 상속 : 부모 클래스의 정보를 물려 받는 것
 
클래스
- 필드, 메서드, 생성자
- 필드 : 속성, 메서드 : 기능, 생성자 : 필드 초기화
- 메서드 선언 방법 : 리턴타입, 매개변수, 메서드명 정리
- 메서드 오버로딩 : 동일한 이름의 메서드가 여러개 있는 것
- 생성자 오버로딩 : 생성자가 여러개
- 생성자
  - 리턴타입이 없고, 클래스명과 동일
- this 객체, this()
  - this 객체 : 메서드에서 필드명과 매개변수명이 같을 때, 구분짓기 위해 사용
  - this() : 생성자에 이미 만들어진 생성자를 이용할 때
- static
  - 모든 객체가 하나의 정보를 이용하거나 기능이 필요할 때 사용
  - 클래스명.필드 또는 클래스명.메서드
  - 객체명.필드(사용가능) => 비추천
- 접근 제어자
  - public : 전부
  - proteced : 나 + 자식 + 같은 패키지
  - 디폴트 : 나 + 같은 패키지
  - private : 나 
- final
  - 변수 : 상수
  - 메서드 : 오버라이딩 금지
  - 클래스 : 상속 금지
- 상속  
  - 부모 필드와 메서드를 물려 받음
  - A는 B이다가 성립하면 A: 자식, B: 부모
- 다형성
  - 부모(조상) 객체가 자식(자손) 클래스의 객체를 관리
  - 객체를 관리 : 객체 다형성
  - 메서드의 매개변수를 관리 : 매개변수의 다형성
    => 메서드 오버로딩이 된 것과 같은 효과
  - 클래스 형변환
    - up 캐스팅
    - down 캐스팅
- 메서드 오버라이딩 : 부모 클래스의 메서드를 재정의

추상 메서드
- 선언부만 있고, 구현부가 없는 메서드
- 실제 기능이 구현 안된 메서드

인터페이스 
- 클래스 상수, 추상 메서드, 디폴트 메서드
- 인터페이스는 구현 클래스를 이용하여 객체를 생성

Object 클래스
- 최상위 조상 클래스
- equals, toString 등을 오버라이딩 하여 사용

제네릭

컬렉션 프레임워크

- Collection 인터페이스 : 하나의 자료형을 관리
  - List : 중복 허용, 순서 보장 => 번지를 통해 접근
  - Set : 중복 X, 순서 보장 X
- Map 인터 페이스
  - 2개의 자료형을 관리
  - Key : 중복 X, Value : 중복 O
